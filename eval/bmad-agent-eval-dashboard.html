Evaluation Methodology

1 Objective Metric

Each metric is a number you can measure, not "feels faster." All 9 metrics have concrete formulas derived from timestamps, counts, and ratios already tracked in your delivery toolchain (Jira, Git, Confluence, ADR logs).

Composite Score = weighted average:
0.35 × Speed + 0.35 × Quality + 0.30 × Coverage

2 Measurement Tool

Automated, reliable, no human needed. All metrics are captured from existing systems: ticket timestamps for speed, PR approval status for quality, section word-counts and item-counts for coverage. The /bmad-eval command auto-collects from .bmad/ artifacts.

Statistical significance via two-sample t-test at p < 0.05 across cohorts.

3 Lever to Pull

Something the agent can change. BMAD agents directly affect: draft generation speed (templates + context), quality (built-in NFR checklists), and coverage (systematic alternatives analysis). Each lever maps to a specific agent skill.

Control group: Practitioners without AI for weeks 1-4, with AI for weeks 5-12. Same projects, same complexity tier.
